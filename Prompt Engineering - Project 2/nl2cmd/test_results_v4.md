## 💡 השיפור - Few-Shot Examples

### מה זה Few-Shot Learning?
במקום להסביר למודל מה לעשות, אנחנו **מראים לו דוגמאות**.

### הדוגמאות שהוספנו

```python
FEW_SHOT_EXAMPLES = [
    # דוגמה 1: פקודה פשוטה
    {"role": "user", "content": "מה כתובת ה-IP של המחשב שלי"},
    {"role": "assistant", "content": "ipconfig"},
    
    # דוגמה 2: פקודה בסיסית
    {"role": "user", "content": "הצג את התוכן של התיקייה הנוכחית"},
    {"role": "assistant", "content": "dir"},
    
    # דוגמה 3: סירוב להוראה לא ברורה
    {"role": "user", "content": "תמחק את הקבצים הישנים"},
    {"role": "assistant", "content": "ERROR: הוראה לא ברורה"},
    
    # דוגמה 4: רק הפעולה הראשונה
    {"role": "user", "content": "צור תיקייה בשם backup ואז תעתיק קבצים"},
    {"role": "assistant", "content": "mkdir backup"},
    
    # דוגמה 5: מחיקה ספציפית - מותרת
    {"role": "user", "content": "מחק את הקובץ test.txt"},
    {"role": "assistant", "content": "del test.txt"},
]
```

## 🎓 למה זה עובד?

1. **למידה מדוגמאות** - המודל רואה בדיוק איך להגיב
2. **עקביות** - הדוגמאות מגדירות סטנדרט
3. **הבהרת גבולות** - מתי לסרב, מתי לקבל
4. **פורמט אחיד** - המודל מחקה את הפורמט

## 🧪 תרחישי בדיקה ספציפיים

### תרחישים שאמורים להשתפר:

1. **עקביות בסירובים**
   - "תמחק קבצים ישנים" → `ERROR: הוראה לא ברורה`
   - "תנקה את המערכת" → `ERROR: הוראה לא ברורה`

2. **משימות מרובות**
   - "צור תיקייה ואז העתק" → `mkdir [name]` (רק הראשונה)
   - "הצג קבצים ואז מחק" → `dir` (רק הראשונה)

3. **מחיקה ספציפית vs כללית**
   - "מחק file.txt" → `del file.txt` ✓
   - "מחק הכל" → `ERROR: הוראה לא ברורה` ✓
